Apa itu Senka Uploader API?

Bayangkan Anda punya aplikasi dan ingin pengguna bisa mengunggah file (gambar, dokumen, video, dll). Senka Uploader API adalah solusi siap pakai yang membuat proses ini jadi sangat mudah!

Cepat & Efisien

Unggah file dalam hitungan detik. Tidak perlu setup rumit, langsung bisa dipakai.

Aman

File Anda terlindungi dengan enkripsi dan keamanan tingkat enterprise.

Skalabel

Dukungan untuk file besar hingga 50MB dan siap untuk skala enterprise.

Cara Mengunggah File

Ini adalah bagian paling penting! Anda akan mengirim file ke server kami, dan kami akan memberikan Anda link untuk mengakses file tersebut.

POST /api/upload

Cara Mengirim File

Gunakan format multipart/form-data - ini adalah format standar untuk mengirim file melalui form.

Form data harus berisi:

  • file: (WAJIB) File yang ingin Anda unggah. Bisa gambar, dokumen, video, atau format lainnya.

Batasan & Tips

Ukuran Maksimal

50MB per file. Jika file Anda lebih besar, coba kompres dulu atau bagi menjadi beberapa bagian.

Format Didukung

Semua format file didukung! JPG, PNG, GIF, PDF, DOC, MP4, MP3, dan banyak lagi.

Contoh 1: Menggunakan cURL (Command Line)

curl_example.sh 
curl -X POST \
  http://mydomain.com/api/upload \
  -H "Content-Type: multipart/form-data" \
  -F "file=@/path/to/your/image.jpg"

Contoh 2: Menggunakan JavaScript (Fetch API)

upload.js 
const uploadFile = async (file) => {
  const formData = new FormData();
  formData.append('file', file);

  try {
    const response = await fetch('http://mydomain.com/api/upload', {
      method: 'POST',
      body: formData
    });

    const result = await response.json();
    
    if (response.ok) {
      console.log('Upload berhasil:', result);
    } else {
      console.error('Upload gagal:', result.message);
    }
  } catch (error) {
    console.error('Terjadi kesalahan:', error);
  }
};

Apa yang Anda Dapatkan Setelah Upload?

Jika upload berhasil, server akan mengembalikan status 200 OK dengan informasi lengkap tentang file Anda.

response_success.json 
{
  "success": true,
  "message": "File berhasil diupload!",
  "fileName": "nama-file-asli.jpg",
  "fileSize": 123456,
  "fileType": "image/jpeg",
  "fileUrl": "https://your-supabase-domain.storage.supabase.co/storage/v1/object/public/your-bucket/unik-filename.jpg",
  "storagePath": "unik-filename.jpg"
}

Jika terjadi error, server akan memberikan pesan yang jelas tentang apa yang salah:

error_no_file.json 
{
  "success": false,
  "message": "Tidak ada file yang diupload.",
  "error": "Error details here (optional)"
}
error_file_size.json 
{
  "success": false,
  "message": "Ukuran file melebihi batas maksimal (50MB).",
  "error": "LIMIT_FILE_SIZE"
}

Cara Melihat atau Mengunduh File yang Sudah Diunggah

Setelah Anda mengunggah file, Anda akan mendapatkan URL unik. Gunakan URL ini untuk menampilkan atau mengunduh file kapan saja!

GET /files/:fileName

Parameter yang Dibutuhkan

  • :fileName: Nama file unik yang Anda dapatkan setelah upload. Contoh: 1701234567890-abcdefg.jpg

Tips Penggunaan

Untuk menampilkan gambar di website Anda, cukup gunakan URL ini sebagai src di tag .

Untuk video, gunakan URL ini di tag . Untuk dokumen, Anda bisa membuat link download.

Contoh 1: Menggunakan cURL

download_curl.sh 
curl -X GET \
  http://localhost:5000/files/1701234567890-abcdefg.jpg

Contoh 2: Menampilkan Gambar di Website

image_tag.html

Apa yang Anda Dapatkan?

Server akan mengirimkan file asli Anda dengan header yang sesuai (misalnya, gambar akan dikirim sebagai image/jpeg).

http_response.txt 
HTTP/1.1 200 OK
Content-Type: image/jpeg
Content-Length: 123456

[Binary file data]

Jika file tidak ditemukan, Anda akan mendapatkan pesan error dalam format JSON.

Penanganan Kesalahan - Jangan Panik!

Kadang-kadang sesuatu tidak berjalan sesuai rencana. Tenang, kami sudah menyiapkan panduan lengkap untuk membantu Anda menyelesaikan masalah!

Kode Error Apa Artinya? Cara Memperbaiki
400 Bad Request Anda lupa mengirim file atau formatnya salah. Pastikan Anda mengirimkan file dalam form data dengan nama field 'file'.
404 Not Found File yang Anda minta tidak ditemukan. Periksa kembali nama file atau pastikan file sudah diupload sebelumnya.
413 Payload Too Large File Anda terlalu besar (lebih dari 50MB). Kompres file Anda atau bagi menjadi beberapa bagian yang lebih kecil.
500 Internal Server Error Ada masalah di server kami (jarang terjadi). Coba lagi beberapa saat kemudian atau hubungi tim pengembang.
